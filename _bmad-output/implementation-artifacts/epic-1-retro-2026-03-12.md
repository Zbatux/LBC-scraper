## What Went Well

- **Architecture quality:** Interface contracts (e.g., `find_match(lat, lng, area, candidates) -> int | None`) were precise and unambiguous. Every story knew exactly which files to touch and which to leave alone.
- **Dependency sequencing:** config → matcher → schema → parsers → save_or_merge — each story built on the previous without blocking.
- **Migration pattern:** Additive `ALTER TABLE ADD COLUMN` with `try/except OperationalError` made schema changes idempotent. Zero crashes on existing databases.
- **Test growth:** 25 → 35 → 53 → 65 tests grew organically with each story. Zero regressions across the entire sprint.
- **Module isolation:** matcher.py has no DB access, parsers.py has no Flask imports — boundaries held at 100%.

## Challenges

### Bug: Falsy Zero (Story 1.3 — Critical)

**Issue:** `if lat and lng:` treated `lat=0.0` as `None`, silently skipping GPS routing for equatorial coordinates.
**Fix:** Changed to `if lat is not None and lng is not None:`.
**Impact:** Would have caused silent data corruption in production — GPS-valid listings treated as GPS-less.
**Detection:** Code review only. Unit tests did not cover `lat=0.0` edge case.

### Bug: Missing Float Cast (Story 1.3 — Medium)

**Issue:** `get_coords()` did not explicitly cast to `float()`, accepting non-numeric types silently.
**Fix:** Added `float()` cast with `ValueError`/`TypeError` guard.
**Detection:** Code review.

### Bug: Missing Change Detection Field (Story 1.4 — Medium)

**Issue:** `trajet` field was not included in change detection logic. A travel time change would not trigger a history snapshot.
**Fix:** Added `"trajet"` to the comparison field list.
**Detection:** Code review.

---

## Key Insights

1. **Code review is the #1 safety net for subtle bugs.** 2 out of 3 bugs were found only by review, not by tests. Unit tests verify expected logic; reviews catch unexpected assumptions.
2. **`is not None` > truthiness for numeric comparisons in Python.** This is a recurring Python pitfall that should be a standard practice rule.
3. **Well-specified architecture eliminates blockers.** When contracts are precise, stories can be implemented independently without coordination overhead.
